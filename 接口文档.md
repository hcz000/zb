# 直播辩论系统 · 后端接口文档

> 适用系统：直播辩论系统（前端 uni-app / H5 + 网关 Gateway + 本后端 Backend）
> 文档状态：后端 Mock 实现版（v1）
> 最后更新：2026-07-13

---

## 1. 概述

### 1.1 系统架构

```
小程序 / H5 前端 (Frontend)
        │  HTTP / WebSocket
        ▼
网关 Gateway (8080)   ← 转发 /api/* 与 /ws，托管 /admin 后台页
        │  HTTP / WebSocket
        ▼
后端 Backend          ← 本服务：Mock 数据 + 全部业务逻辑
```

本接口文档描述的是 **Backend（后端）** 对外暴露的接口。网关仅做转发与静态托管，不内置业务数据。

### 1.2 基础信息

| 项 | 值 |
| --- | --- |
| Base URL（本地） | `http://localhost:3000` |
| Base URL（经网关） | `http://localhost:8080/api/*`（网关代理到后端） |
| 协议 | HTTP + WebSocket（`/ws`） |
| 数据格式 | `application/json`（请求与响应） |
| 跨域 | 开发环境 `Access-Control-Allow-Origin: *` |

### 1.3 统一响应格式

为满足「前端可直接联调」，后端统一返回如下结构（与现有前端解析逻辑一致）：

```json
{
  "success": true,
  "data": { },
  "message": "操作成功",
  "timestamp": 1752384000000
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | boolean | 业务是否成功 |
| `data` | object/array | 业务数据载体 |
| `message` | string | 提示信息（失败时为错误原因） |
| `timestamp` | number | 服务端毫秒时间戳 |

> 说明：需求文档示例给出的 `{ code, message, data }` 亦可作为兼容外壳再包一层；本实现优先保证与现有前端对接可用，故采用 `success` 信封。错误时 HTTP 状态码随语义返回（`400`/`404`/`409`/`500`），并附带 `success:false` + `message`。

### 1.4 鉴权说明

- 公开接口无需登录即可调用（演示用）。
- 微信登录 `POST /api/wechat-login` 采用 **Mock 模式**：不真正请求微信服务器，直接返回模拟 `openid` / `session_key`。
- 如需接真微信，将后端配置 `wechat.useMock` 置为 `false` 并填入 `appid` / `secret` 即可（接口契约不变）。
- 管理类接口（`/api/admin/*`）当前未强制鉴权，预留 `Authorization` 头透传位。

## 3. 接口列表

### 3.1 鉴权与用户

#### POST `/api/wechat-login`
微信登录（Mock 模式）。
- **请求体**
```json
{ "code": "081abc...", "userInfo": { "nickName": "小明", "avatarUrl": "https://..." } }
```
- **响应**
```json
{
  "success": true,
  "data": {
    "openid": "mock_openid_1752384000000",
    "session_key": "mock_session_key_xxxx",
    "unionid": null,
    "userInfo": { "nickName": "小明", "avatarUrl": "https://..." },
    "loginTime": "2026-07-13T10:00:00.000Z",
    "isMock": true
  }
}
```

#### GET `/api/admin/users`
获取全部用户列表。
- **响应** `data`: `User[]`

#### GET `/api/admin/users/:id`
获取单个用户。不存在返回 `404 { success:false, error:"用户不存在" }`。

#### GET `/api/admin/miniprogram/users`
小程序在线用户列表（分页 + 排序）。
- **Query**: `page=1` `pageSize=20` `status=all|online` `orderBy=joinTime|votes`
- **响应** `data`: `{ total, page, pageSize, users: [...] }`

---

### 3.2 辩题

#### GET `/api/debate-topic`
获取当前辩题（前端调用）。
- **响应** `data`: `DebateTopic`

#### GET `/api/admin/debate`
获取辩题设置（后台）。
- **响应** `data`: `DebateTopic`

#### PUT `/api/admin/debate`
更新辩题设置。
- **请求体**: `{ "title": "...", "description": "...", "leftPosition":"正方", "rightPosition":"反方" }`
- **响应** `data`: 更新后的 `DebateTopic`（同时触发 `debate-updated` WS 广播）

---

### 3.3 投票

#### GET `/api/votes`
获取实时票数（公开）。
- **响应** `data`: `Votes`

#### POST `/api/user-vote`
用户投票，支持两种格式：
- 格式一（增量）：`{ "side": "left"|"right", "votes": 10 }`
- 格式二（100 票分配）：`{ "leftVotes": 60, "rightVotes": 40 }`（两值和须为 100）
- 可选 `userId` 用于统计。
- **响应** `data`: `Votes` + `message`（触发 `votes-updated` WS 广播）

#### GET `/api/admin/votes`
后台获取票数（同 `Votes` 结构）。

#### PUT `/api/admin/votes`
后台直接修改票数。
- **请求体**: `{ "leftVotes": 100, "rightVotes": 80 }`（字段可缺省）

#### POST `/api/admin/votes/reset`
重置票数（清零）。

#### POST `/api/admin/live/update-votes`
直播中更新票数：`{ "action": "set|add|reset", "leftVotes": 0, "rightVotes": 0 }`

#### POST `/api/admin/live/reset-votes`
直播中重置票数：`{ "resetTo": { "leftVotes":0, "rightVotes":0 }, "saveBackup": true }`

#### GET `/api/admin/votes/statistics`
投票统计（含时间轴）：`data: { summary, timeline, topVoters }`

---

### 3.4 AI 观点与互动

#### GET `/api/ai-content`
获取全部 AI 观点（公开）。`data`: `AIContent[]`

#### GET `/api/admin/ai-content`
同公开列表（后台用）。

#### GET `/api/admin/ai-content/list`
分页列表：`?page=1&pageSize=20&startTime=&endTime=`
- **响应** `data`: `{ total, page, pageSize, items: AIContent[] }`

#### GET `/api/v1/admin/ai-content/list`
v1 版本列表（同上，返回字段更规整：`type/position/confidence/statistics`）。

#### GET `/api/admin/ai-content/:id`
单条观点详情。不存在返回 `404`。

#### POST `/api/admin/ai-content`
新增观点：`{ "text": "...", "side": "left|right", "debate_id": "..." }`
- 触发 `newAIContent` WS 广播。

#### PUT `/api/admin/ai-content/:id`
更新观点：`{ "text": "...", "side": "left|right" }`

#### DELETE `/api/admin/ai-content/:id`
删除观点。

#### GET `/api/admin/ai-content/:id/comments`
观点评论列表：`?page=1&pageSize=20` → `data: { contentId, contentText, total, comments }`

#### GET `/api/v1/admin/ai-content/:id/comments`
v1 评论列表（按时间倒序，字段规范化）。

#### DELETE `/api/admin/ai-content/:id/comments/:commentId`
删除观点下某条评论：`{ "reason": "...", "notifyUsers": true }`

#### DELETE `/api/v1/admin/ai-content/:id/comments/:commentId`
v1 删除评论（触发 `comment-deleted` WS 广播）。

#### POST `/api/comment`
新增评论：`{ "contentId": "uuid", "user": "小明", "text": "说得对", "avatar": "👤" }`
- **响应** `data`: 新评论对象

#### DELETE `/api/comment/:commentId`
删除评论：`{ "contentId": "uuid" }`

#### POST `/api/like`
点赞（内容或评论）：
```json
{ "contentId": "uuid", "commentId": "可选" }
```
- **响应** `data`: `{ "likes": 46 }`

### 3.5 直播控制

#### POST `/api/live/control`
公开控制直播：`{ "action": "start|stop", "streamId": "可选" }`
- 触发 `live-status-changed` / `liveStatus` WS 广播。

#### POST `/api/admin/live/control`
后台控制直播（同结构，管理员专用）。

#### POST `/api/admin/live/start`
开始直播：`{ "streamId": "可选", "autoStartAI": false, "notifyUsers": true }`
- 返回 `data`: `{ liveId, streamUrl, status, startTime }`

#### POST `/api/admin/live/stop`
停止直播：`{ "streamId": "可选", "saveStatistics": true }`
- 返回 `data`: `{ liveId, status, stopTime, duration, summary }`

#### POST `/api/admin/live/setup-and-start`
一键设置并开始：`{ "streamId", "scheduledStartTime", "scheduledEndTime", "startNow": true }`

#### GET `/api/admin/live/status`
获取直播状态 + 当前启用流的播放地址。

#### POST `/api/admin/live/schedule`
设置直播计划：`{ "scheduledStartTime", "scheduledEndTime", "streamId" }`

#### GET `/api/admin/live/schedule`
获取直播计划。

#### POST `/api/admin/live/schedule/cancel`
取消直播计划。

---

### 3.6 直播流管理

#### GET `/api/admin/streams`
直播流列表（含 `playUrls` 与 `liveStatus`）。

#### POST `/api/admin/streams`
新增流：`{ "name", "url", "type": "hls|rtmp|flv", "description", "enabled": true }`
- `name/url/type` 必填，URL 需合法。

#### PUT `/api/admin/streams/:id`
更新流信息（字段可部分更新）。

#### DELETE `/api/admin/streams/:id`
删除流（正在直播中则返回 `400`）。

---

### 3.7 AI 识别控制

#### POST `/api/admin/ai/start`
启动 AI 识别：`{ "settings": { "mode":"realtime", "interval":5000, "sensitivity":"high", "minConfidence":0.7 } }`

#### POST `/api/admin/ai/stop`
停止 AI 识别：`{ "saveHistory": true }`

#### POST `/api/admin/ai/toggle`
暂停/恢复：`{ "action": "pause|resume" }`

#### DELETE `/api/admin/ai/content/:contentId`
删除 AI 内容（触发 `aiContentDeleted` 广播）。

## 4. WebSocket 实时通信

- **连接地址**：`ws://<host>/ws`（经网关同路径）
- **连接成功**服务端先推送：
```json
{ "type": "connected", "message": "已连接到实时数据服务" }
```
- 新连接还会收到当前状态快照 `type: "state"`。

### 4.1 客户端 → 服务端 消息

| type | 载荷 | 说明 |
| --- | --- | --- |
| `ping` | — | 心跳，服务端回 `pong` |
| `control-live` | `{ "action": "start|stop" }` | 后台控制直播 |
| `update-debate` | `{ "debate": {...} }` | 更新辩题（实时通知） |

### 4.2 服务端 → 客户端 推送

| type | 含义 |
| --- | --- |
| `live-status-changed` / `liveStatus` | 直播开始/停止状态变化 |
| `votes-updated` | 票数更新（含百分比、用户投票明细） |
| `debate-updated` | 辩题更新 |
| `newAIContent` | 新增 AI 观点 |
| `aiContentDeleted` | AI 观点被删除 |
| `comment-deleted` | 评论被删除 |
| `aiStatus` | AI 识别运行状态变化 |
| `live-schedule-updated` / `live-schedule-cancelled` | 直播计划变化 |
| `state` | 新连接时的状态快照 |

## 5. 错误码约定

| HTTP | 场景 |
| --- | --- |
| 400 | 参数缺失/格式错误（如 100 票分配和不为 100） |
| 404 | 资源不存在（用户/观点/评论/流） |
| 409 | 状态冲突（如直播已在进行、AI 已运行） |
| 500 | 服务端内部错误（返回 `success:false` + `message`） |

## 6. 部署相关（网关转发映射）

网关需将以下路径代理到本后端（默认后端端口 `3000`）：

| 网关入口 | 转发目标 |
| --- | --- |
| `/api/*` | `http://backend:3000/api/*` |
| `/ws` | `ws://backend:3000/ws` |

网关自身保留：`/admin`（后台静态页）、健康检查。
