## 1. 系统架构与多流模型

### 1.1 部署架构

```
小程序 / H5 前端 (Frontend)
        │  HTTP / WebSocket
        ▼
网关 Gateway (8080)   ← 转发 /api/* 与 /ws，托管 /admin 后台页
        │  HTTP / WebSocket
        ▼
后端 Backend          ← 本服务：Mock 数据 + 全部业务逻辑（多流）
```

- 网关仅做转发与静态托管，**不内置业务数据**（生产形态）。
- 当前 `server.js` / `gateway.js` 为「网关 + 后端同进程」实现，本地调试可直接运行。

### 1.2 多直播流（Multi-Stream）核心约定

系统支持**同时运行多场独立直播辩论**，每场称为一个「直播流（Stream）」，由 `streamId` 唯一标识。

每个直播流拥有**完全独立**的数据：

- 独立的票数数据（正方 / 反方票数）
- 独立的辩题（可共用默认辩题，也可按流覆盖）
- 独立的评委配置（3 位评委）
- 独立的辩论流程配置（环节、时长、顺序）
- 独立的流程控制状态（计时器、当前环节、剩余时间）
- 独立的直播运行状态

**强制规则（所有接口与事件必须遵守）：**

1. 写操作（`POST/PUT/DELETE`）涉及具体流时，请求体需携带 `streamId`（或路径/查询参数 `stream_id`）。
2. 读操作列表类接口支持可选 `stream_id` 查询参数，用于按流过滤；不传则返回全部。
3. 所有 WebSocket 推送事件 `data` 中**必须包含 `streamId`** 字段。
4. 前端（数据大屏 / 小程序）按 `streamId` **严格过滤**消息，只处理当前所选流的消息。

## 2. 基础约定

### 2.1 数据格式

- 请求与响应均为 `application/json`。
- 开发环境跨域：`Access-Control-Allow-Origin: *`。

### 2.2 基础地址

| 环境 | Base URL |
| --- | --- |
| 本地（经网关） | `http://localhost:8080` |
| 本地（直连后端） | `http://localhost:3000` |
| WebSocket | `ws://<host>:8080/ws` |
| 路径前缀 | 所有接口以 `/api` 开头（兼容 `/api/v1` 别名） |

### 2.3 统一响应格式

```json
{
  "success": true,
  "code": 0,
  "message": "操作成功",
  "data": { },
  "timestamp": 1752384000000
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | boolean | 业务是否成功（前端以此判断） |
| `code` | number | 状态码，`0`=成功，非 0=业务错误（兼容需求文档示例） |
| `message` | string | 提示信息（失败时为错误原因） |
| `data` | object/array | 业务数据载体 |
| `timestamp` | number | 服务端毫秒时间戳 |

> 成功时 `code=0 && success=true`；失败时 `success=false` 且 `code` 取对应错误码（见 §6），HTTP 状态随语义返回。

### 2.4 多流参数约定

| 参数 | 位置 | 类型 | 说明 |
| --- | --- | --- | --- |
| `stream_id` | query / body | string | 直播流 ID。写操作必填；读列表可选（过滤用） |
| `streamId` | body / WS data | string | 同上，JSON 字段命名（与 `stream_id` 等价，后端自动兼容） |

### 2.5 鉴权说明

- 公开接口无需登录即可调用（演示用）。
- 微信登录 `POST /api/wechat-login` 采用 **Mock 模式**：直接返回模拟 `openid` / `session_key`。
- 管理类接口（`/api/admin/*`）预留 `Authorization` 头透传位，当前未强制鉴权。

## 3. 数据模型（全部按流隔离）

### 3.1 Stream（直播流）

```json
{
  "id": "stream-001",
  "name": "主直播间",
  "url": "rtmp://...",
  "type": "rtmp",
  "description": "",
  "enabled": true,
  "createdAt": "2026-07-13T10:00:00.000Z",
  "playUrls": { "hls": null, "flv": null, "rtmp": "rtmp://..." },
  "liveStatus": { "isLive": false, "liveId": null, "startTime": null, "streamId": "stream-001" }
}
```

### 3.2 DebateTopic（辩题）

```json
{
  "id": "debate-default-001",
  "streamId": "stream-001",
  "title": "如果有一个能一键消除痛苦的按钮，你会按吗？",
  "description": "这是一个关于痛苦、成长与人性选择的深度辩论",
  "leftPosition": "正方",
  "rightPosition": "反方"
}
```

### 3.3 Votes（票数，按流存储）

```json
{
  "streamId": "stream-001",
  "leftVotes": 120,
  "rightVotes": 88,
  "totalVotes": 208,
  "leftPercentage": 58,
  "rightPercentage": 42
}
```

> 后端存储结构：`streamVotes = { "stream-001": {...}, "stream-002": {...} }`，每流独立累加。

### 3.4 AIContent（AI 观点）

```json
{
  "id": "uuid",
  "streamId": "stream-001",
  "debate_id": "debate-default-001",
  "text": "正方观点：痛苦是人生成长的必要经历……",
  "side": "left",
  "timestamp": 1752383700000,
  "likes": 45,
  "comments": [
    {
      "id": "uuid",
      "user": "心理学家",
      "text": "痛苦确实能促进心理成长",
      "time": "3分钟前",
      "avatar": "🧠",
      "likes": 15
    }
  ]
}
```

### 3.5 User（用户）

```json
{
  "id": "openid_xxx",
  "nickname": "微信用户",
  "avatar": "👤",
  "createdAt": "2026-07-13T10:00:00.000Z",
  "statistics": { "totalVotes": 0, "totalComments": 0, "totalLikes": 0 }
}
```

### 3.6 Judge（评委，按流配置）

```json
{
  "id": "judge-1",
  "streamId": "stream-001",
  "name": "张教授",
  "role": "主评委",
  "avatar": "data:image/png;base64,...",
  "leftVotes": 60,
  "rightVotes": 40
}
```

> 每个流固定 3 位评委；评委票数（`leftVotes+rightVotes=100`）单独统计，不影响普通用户票数。

### 3.7 DebateFlow（辩论流程，按流配置）

```json
{
  "streamId": "stream-001",
  "flow": [
    { "name": "正方发言", "duration": 180, "side": "left", "order": 1 },
    { "name": "反方质问", "duration": 120, "side": "right", "order": 2 }
  ],
  "updatedAt": "2026-07-13T10:30:00.000Z"
}
```

> `side` 取值：`left` / `right` / `both`；`order` 从 1 连续递增；`duration` 单位秒。

## 4. 接口列表

> 约定：`{streamId}` 表示路径参数；`?stream_id=` 表示查询参数；请求体字段未标注必填者均可选。

### 4.1 鉴权与用户

#### POST `/api/wechat-login`
微信登录（Mock）。
- 请求：`{ "code": "081abc...", "userInfo": { "nickName": "小明", "avatarUrl": "https://..." } }`
- 响应 `data`：`{ "openid": "...", "session_key": "...", "unionid": null, "userInfo": {...}, "loginTime": "...", "isMock": true }`

#### GET `/api/admin/users`
获取全部用户列表。`data`: `User[]`

#### GET `/api/admin/users/:id`
获取单个用户；不存在返回 `404`。

#### GET `/api/admin/miniprogram/users`
小程序在线用户（分页 + 排序）。
- Query：`page=1` `pageSize=20` `status=all|online` `orderBy=joinTime|votes`
- 响应 `data`：`{ total, page, pageSize, users: [...] }`

---

### 4.2 直播流管理（多流基础）

#### GET `/api/admin/streams`
直播流列表（含 `playUrls` 与 `liveStatus`）。`data`: `Stream[]`

#### POST `/api/admin/streams`
新增流：`{ "name", "url", "type": "hls|rtmp|flv", "description", "enabled": true }`（`name/url/type` 必填，URL 需合法）。

#### PUT `/api/admin/streams/:id`
更新流信息（字段可部分更新）。

#### DELETE `/api/admin/streams/:id`
删除流（该流正在直播中返回 `400`）。

---

### 4.3 辩题（多流）

#### GET `/api/debate-topic?stream_id=`
获取当前辩题（前端调用）。`data`: `DebateTopic`

#### GET `/api/admin/debate?stream_id=`
获取辩题设置（后台）。`data`: `DebateTopic`

#### PUT `/api/admin/debate`
更新辩题设置：`{ "streamId": "stream-001", "title": "...", "description": "...", "leftPosition":"正方", "rightPosition":"反方" }`
- 触发 `debate-updated` WS 广播（带 `streamId`）。

### 4.4 投票（多流核心）

票数数据按 `streamVotes[streamId]` 独立存储，互不影响。

#### GET `/api/votes?stream_id=`
获取实时票数（公开）。`data`: `Votes`

#### POST `/api/user-vote`
用户投票，支持两种格式（均需带 `streamId`）：
- 增量：`{ "streamId": "stream-001", "side": "left"|"right", "votes": 10 }`
- 100 票分配：`{ "streamId": "stream-001", "leftVotes": 60, "rightVotes": 40 }`（两值和须为 100）
- 兼容包装格式：`{ "request": { ... } }`
- 可选 `userId` 用于统计。
- 触发 `votes-updated` WS 广播（带 `streamId` + `userVote`）。

#### GET `/api/admin/votes?stream_id=`
后台获取票数。`data`: `Votes`

#### PUT `/api/admin/votes`
后台直接修改票数：`{ "streamId": "stream-001", "leftVotes": 100, "rightVotes": 80 }`（字段可缺省）。

#### POST `/api/admin/votes/reset`
重置票数（清零）：`{ "streamId": "stream-001" }`

#### POST `/api/admin/live/update-votes`
直播中更新票数：`{ "streamId": "stream-001", "action": "set|add|reset", "leftVotes": 0, "rightVotes": 0 }`
- 触发 `votes-updated` WS 广播。

#### POST `/api/admin/live/reset-votes`
直播中重置票数：`{ "streamId": "stream-001", "resetTo": { "leftVotes":0, "rightVotes":0 }, "saveBackup": true }`

#### GET `/api/admin/votes/statistics?stream_id=`
投票统计（含时间轴）：`data`: `{ summary, timeline, topVoters }`

### 4.6 直播控制（多流）

直播状态按 `streamLiveStatuses[streamId]` 独立维护。

## 5. WebSocket 实时通信（多流）

- 连接地址：`ws://<host>:8080/ws`
- 连接成功服务端先推送：`{ "type": "connected", "message": "已连接到实时数据服务" }`
- 新连接收到状态快照：`{ "type": "state", "data": { ... } }`

### 5.1 客户端 → 服务端

| type | 载荷 | 说明 |
| --- | --- | --- |
| `ping` | — | 心跳，服务端回 `pong` |
| `control-live` | `{ "action": "start|stop", "streamId": "..." }` | 后台控制直播 |
| `update-debate` | `{ "debate": {...}, "streamId": "..." }` | 更新辩题（实时通知） |

### 5.2 服务端 → 客户端（均带 `streamId`）

| type | 含义 |
| --- | --- |
| `live-status-changed` / `liveStatus` | 直播开始/停止（含 `streamId`） |
| `votes-updated` | 票数更新（含 `streamId`、`userVote`） |
| `debate-updated` | 辩题更新（含 `streamId`） |
| `newAIContent` | 新增 AI 观点（含 `streamId`） |
| `aiContentDeleted` | AI 观点被删除（含 `streamId`） |
| `comment-deleted` | 评论被删除（含 `streamId`） |
| `aiStatus` | AI 识别状态变化（含 `streamId`） |
| `judges-updated` | 评委配置更新（含 `streamId`）★★ 待实现 |
| `debate-flow-updated` | 流程配置更新（含 `streamId`）★★ 待实现 |
| `debate-flow-control` | 流程控制命令（含 `streamId`）★★ 待实现 |
| `live-schedule-updated` / `live-schedule-cancelled` | 直播计划变化（含 `streamId`） |
| `state` | 新连接状态快照 |

### 5.3 前端多流消息过滤规则

```javascript
const messageStreamId = message.data.streamId;
const currentStreamId = getCurrentStreamId(); // 当前所选直播流

// 严格过滤：只处理匹配的流；无 streamId 视为全局消息
if (messageStreamId && messageStreamId !== currentStreamId) {
  return; // 忽略其他流的消息
}
if (!messageStreamId || messageStreamId === currentStreamId) {
  updateDisplay(message.data);
}
```

## 6. 错误码约定

| HTTP | code | 场景 |
| --- | --- | --- |
| 400 | 400 | 参数缺失/格式错误（如 100 票分配和不为 100；流正在直播中删除） |
| 404 | 404 | 资源不存在（用户/观点/评论/流/stream_id 不存在） |
| 409 | 409 | 状态冲突（如直播已在进行、AI 已运行） |
| 500 | 500 | 服务端内部错误（`success:false` + `message`） |

错误响应示例：
```json
{ "success": false, "code": 404, "message": "用户不存在", "data": null, "timestamp": 1752384000000 }
```
